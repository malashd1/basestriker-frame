## How it works

When a Farcaster cast links to **`https://basestriker.xyz`**, Warpcast / Coinbase Wallet / any Farcaster client fetches the page, reads the `fc:frame` meta tag, and renders an interactive embed:

```
┌──────────────────────────────────┐
│  🎮 BASESTRIKER                  │
│  Galaxian-style arcade. Base.    │
│                                  │
│  [ ▶ Play ]                      │   ← native button inside the cast
└──────────────────────────────────┘
```

Tapping **Play** opens `https://app.basestriker.xyz` inside the Farcaster client's webview. Wallet is auto-connected via Frame SDK (no separate sign-in).

## Spec: Frame V2 (a.k.a. Mini App)

### `fc:frame` meta tag

Embedded in the `<head>` of `https://basestriker.xyz`:

```html
<meta name="fc:frame" content='{
  "version": "next",
  "imageUrl": "https://basestriker.xyz/frame-image.png",
  "button": {
    "title": "▶ Play",
    "action": {
      "type": "launch_frame",
      "name": "BaseStriker",
      "url": "https://app.basestriker.xyz",
      "splashImageUrl": "https://basestriker.xyz/logo.png",
      "splashBackgroundColor": "#0a0014"
    }
  }
}' />
```

**Tag fields:**

| Field | Value |
|---|---|
| `version` | `"next"` (Frame v2 / Mini App spec) |
| `imageUrl` | 3:2 ratio PNG, 1200×800. Shown in the cast preview. |
| `button.title` | Caption on the action button (max ~32 chars) |
| `action.type` | `launch_frame` — opens the URL in the Farcaster webview |
| `action.url` | The actual app entry — for us, `app.basestriker.xyz` |
| `action.splashImageUrl` | Shown for ~500ms while the webview loads |
| `action.splashBackgroundColor` | Brand colour behind the splash |

### `farcaster.json` manifest

Served at **`https://basestriker.xyz/.well-known/farcaster.json`** — required for Warpcast directory listing + Mini App discoverability.

See [`landing/.well-known/farcaster.json`](landing/.well-known/farcaster.json) for the live version.

⚠️ `accountAssociation` is intentionally empty until a Farcaster account (FID) is registered. Once the operator has an FID, they sign the manifest and paste `{header, payload, signature}` into the empty fields. See [Warpcast docs](https://miniapps.farcaster.xyz/docs/guides/publishing) for the signing flow.

### Preview image

[`landing/frame-image.png`](landing/frame-image.png) — 1200×800 PNG, generated by [`scripts/gen-frame-image.mjs`](scripts/gen-frame-image.mjs). Re-render after gameplay/branding changes:

```bash
npm install sharp
node scripts/gen-frame-image.mjs
```

## Why this matters

- **Distribution surface** — any post in `/base`, `/onchain-games`, `/farcaster-games` channels with a link to basestriker.xyz becomes a one-tap "play now" embed
- **Zero friction** — no app store, no install, no wallet popup before the game loads (wallet is provided by the Farcaster client's host)
- **On-chain footprint** — every shop USDC payment from a Farcaster player still goes through `BaseStrikerPaymentRouter` ([`0xc08bda33…`](https://basescan.org/address/0xc08bda33E32Da9255f21BB57afF78e6d1EAb6789)) and emits an `ItemPaid` event — visible to Talent Protocol, The Graph, Basescan
